## Common structure

Each extension document uses the same sections:

1. **Summary** — what the extension does
2. **What it adds** — commands, tools, shortcuts, or hooks
3. **How it works** — important implementation behavior
4. **Usage** — when to use it
5. **Examples** — practical examples
6. **Files and state** — any files, directories, or persisted state it uses
7. **Notes / caveats** — limitations, assumptions, or current quirks
